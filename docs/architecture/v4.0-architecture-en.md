# uHorse 4.0 Architecture Documentation

## Overview

uHorse 4.0 adopts **Hub-Node distributed architecture**, enabling cloud-local collaborative AI agent system.

### Architecture Diagram


## 1. Core Components

### 1.1 uhorse-protocol

Communication protocol between Hub and Node.

**Message Types**:
- `HubToNode`: Messages from Hub to Node
- `NodeToHub`: Messages from Node to Hub

**Command System**:
- `Shell`: Shell command execution
- `File`: File operations (read/write/edit/delete)
- `Edit`: Code editing
- `Write`: File writing
- `Task`: Task management
- `LLM`: LLM operations
- `Info`: System information

**Binary Codec**:
- Efficient binary encoding
- JSON serialization support
- Compression support

### 1.2 uhorse-node

Local node component.

**Core Features**:
- Task execution engine
- File system operations
- Shell command execution
- Workspace management
- Hub connection management
- Heartbeat and health reporting

**Configuration**:
```rust
NodeConfig {
    node_id: "node-001",
    node_name: "Development Node",
    hub_url: "ws://localhost:8765",
    workspace: WorkspaceInfo {
        name: "main",
        path: "/home/user/workspace",
        read_only: false,
    },
    max_concurrent_tasks: 10,
    heartbeat_interval_secs: 30,
}
```

### 1.3 uhorse-hub

Cloud Hub component.

**Core Features**:
- Multi-node management
- Task scheduling and routing
- Load balancing
- Health monitoring
- Statistics collection

**Configuration**:
```rust
HubConfig {
    hub_id: "hub-001",
    bind_address: "0.0.0.0",
    port: 8765,
    max_nodes: 100,
    heartbeat_timeout_secs: 30,
    task_timeout_secs: 300,
    max_retries: 3,
}
```

## 2. Communication Protocol

### 2.1 Connection Lifecycle

```
Node                              Hub
  │                                │
  │──── Register ─────────────────>│
  │<─── RegisterAck ───────────────│
  │                                │
  │──── Heartbeat ────────────────>│ (every 30s)
  │<─── HeartbeatAck ──────────────│
  │                                │
  │<─── TaskAssignment ────────────│
  │──── TaskAccepted ─────────────>│
  │                                │
  │──── TaskProgress ─────────────>│
  │                                │
  │──── TaskCompleted ────────────>│
  │                                │
  │<─── HeartbeatRequest ──────────│ (on demand)
  │──── Heartbeat ────────────────>│
```

### 2.2 Task Execution Flow

```
User                Hub                 Node
  │                  │                    │
  │── Submit Task ──>│                    │
  │                  │── TaskAssignment ─>│
  │                  │<── TaskAccepted ───│
  │                  │                    │
  │                  │<── TaskProgress ───│
  │                  │<── TaskProgress ───│
  │                  │<── TaskCompleted ──│
  │<── Task Result ──│                    │
```

### 2.3 Priority System

| Priority | Value | Use Case |
|----------|-------|----------|
| Background | 0 | Cleanup, maintenance |
| Low | 1 | Batch processing |
| Normal | 2 | Regular tasks |
| High | 3 | Important tasks |
| Urgent | 4 | Time-sensitive |
| Critical | 5 | System critical |

## 3. Security

### 3.1 JWT Authentication

```rust
// Create authenticator
let authenticator = NodeAuthenticator::with_secret("jwt-secret")?;

// Authenticate node
let auth_info = authenticator.authenticate_node(&node_id, "credentials").await?;

// Verify token
let verified = authenticator.verify_token(&token).await?;

// Refresh token
let new_tokens = authenticator.refresh_token(&refresh_token).await?;
```

### 3.2 Sensitive Operation Approval

```rust
// Create approver
let approver = SensitiveOperationApprover::new(approval_manager);

// Check if approval required
if approver.requires_approval("file_delete") {
    // Request approval
    let request_id = approver.request_approval(
        &node_id,
        "file_delete",
        ApprovalLevel::Single,
        context,
    ).await?;

    // Wait for approval
    let status = approver.get_approval_status(&request_id).await?;
}

// Approve/Reject
approver.approve(&request_id, "admin", Some("Approved")).await?;
approver.reject(&request_id, "admin", Some("Risk too high")).await?;
```

### 3.3 Field Encryption

```rust
// Create encryptor
let encryptor = HubFieldEncryptor::with_master_key(master_key);

// Encrypt data
let encrypted = encryptor.encrypt(b"sensitive data")?;

// Decrypt data
let decrypted = encryptor.decrypt(&encrypted)?;

// Encrypt JSON
let encrypted_json = encryptor.encrypt_json(&json_data)?;
let decrypted_json: Value = encryptor.decrypt_json(&encrypted_json)?;
```

### 3.4 TLS Configuration

```rust
// Create TLS config
let tls_config = HubTlsConfig::new("/path/to/cert.pem", "/path/to/key.pem");

// Use in Hub
let manager = SecurityManager::new("jwt-secret", approval_manager)?
    .with_tls("/etc/ssl/cert.pem", "/etc/ssl/key.pem");
```

## 4. Configuration

### 4.1 Hub Configuration File

`hub.toml`:

```toml
[hub]
id = "hub-001"
bind_address = "0.0.0.0"
port = 8765
max_nodes = 100
heartbeat_timeout_secs = 30
task_timeout_secs = 300
max_retries = 3

[security]
jwt_secret = "your-secret-key"
tls_cert = "/etc/ssl/cert.pem"
tls_key = "/etc/ssl/key.pem"

[storage]
type = "sqlite"
path = "/var/lib/uhorse/hub.db"

[logging]
level = "info"
format = "json"
output = "/var/log/uhorse/hub.log"
```

### 4.2 Node Configuration File

`node.toml`:

```toml
[node]
id = "node-001"
name = "Development Node"
hub_url = "ws://localhost:8765"
max_concurrent_tasks = 10
heartbeat_interval_secs = 30

[workspace]
name = "main"
path = "/home/user/workspace"
read_only = false
allowed_patterns = ["*"]
denied_patterns = []

[logging]
level = "info"
output = "/var/log/uhorse/node.log"
```

## 5. API Reference

### 5.1 Hub HTTP API

#### Node Management

```
GET    /api/v1/nodes              # Get all nodes
GET    /api/v1/nodes/:id          # Get node details
POST   /api/v1/nodes              # Register node
DELETE /api/v1/nodes/:id          # Unregister node
```

#### Task Management

```
GET    /api/v1/tasks              # Get all tasks
GET    /api/v1/tasks/:id          # Get task details
POST   /api/v1/tasks              # Submit task
DELETE /api/v1/tasks/:id          # Cancel task
```

#### Statistics

```
GET    /api/v1/stats              # Get system stats
GET    /api/v1/health             # Health check
```

### 5.2 WebSocket Protocol

WebSocket endpoint: `ws://hub-host:8765/ws`

#### Node → Hub Messages

| Message Type | Description |
|-------------|-------------|
| `Register` | Node registration |
| `Heartbeat` | Heartbeat |
| `TaskAccepted` | Task accepted |
| `TaskProgress` | Task progress |
| `TaskCompleted` | Task completed |
| `TaskFailed` | Task failed |

#### Hub → Node Messages

| Message Type | Description |
|-------------|-------------|
| `RegisterAck` | Registration acknowledgment |
| `HeartbeatAck` | Heartbeat acknowledgment |
| `TaskAssignment` | Task assignment |
| `TaskCancel` | Cancel task |

## 7. Troubleshooting

### 7.1 Common Issues

#### Node Cannot Connect to Hub

```bash
# Check if Hub is running
curl http://localhost:8765/health

# Check network connectivity
telnet localhost 8765

# Check logs
tail -f /var/log/uhorse/hub.log
```

#### Task Execution Failure

```bash
# View task details
uhorse task get <task-id>

# View node logs
tail -f /var/log/uhorse/node.log
```

### 7.2 Debug Mode

Enable debug logging:

```bash
RUST_LOG=uhorse_hub=debug,uhorse_node=debug uhorse-hub
```

## 8. Best Practices

### 8.1 Deployment Recommendations

- Deploy Hub on stable server environment
- Deploy Node on user's local machine
- Use TLS for encrypted communication
- Configure heartbeat interval to 30 seconds
- Set reasonable task timeouts

### 8.2 Performance Optimization

- Adjust `max_concurrent_tasks` based on machine capabilities
- Use SSD for database storage
- Configure task priorities appropriately
- Monitor resource usage

### 8.3 Security Recommendations

- Enable JWT authentication
- Use TLS encryption
- Rotate keys regularly
- Audit sensitive operations
- Restrict network access
