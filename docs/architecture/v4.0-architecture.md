# uHorse 4.0 架构文档

## 概述

uHorse 4.0 采用 **Hub-Node 分布式架构**，实现云端-本地协作的 AI 智能体系统。

### 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                         云端层                               │
│  ┌─────────────────────────────────────────────────────────┐│
│  │                      uHorse Hub                          ││
│  │  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐        ││
│  │  │  Scheduler  │ │  Router     │ │  Monitor    │        ││
│  │  └─────────────┘ └─────────────┘ └─────────────┘        ││
│  │  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐        ││
│  │  │ Node Manager│ │ Task Queue  │ │ Stats       │        ││
```

## 1. 核心组件

### 1.1 uhorse-protocol

Hub 与 Node 之间的通信协议。

**消息类型**：
- `HubToNode`: Hub → Node 的消息
- `NodeToHub`: Node → Hub 的消息

**命令系统**：
- `Shell`: Shell 命令执行
- `File`: 文件操作 (读取/写入/编辑/删除)
- `Edit`: 代码编辑
- `Write`: 文件写入
- `Task`: 任务管理
- `LLM`: LLM 操作
- `Info`: 系统信息

**二进制编解码**：
- 高效的二进制编码
- JSON 序列化支持
- 压缩支持

### 1.2 uhorse-node

本地节点组件。

**核心功能**：
- 任务执行引擎
- 文件系统操作
- Shell 命令执行
- 工作空间管理
- Hub 连接管理
- 心跳与健康报告

**配置**：
```rust
NodeConfig {
    node_id: "node-001",
    node_name: "开发节点",
    hub_url: "ws://localhost:8765",
    workspace: WorkspaceInfo {
        name: "main",
        path: "/home/user/workspace",
        read_only: false,
    },
    max_concurrent_tasks: 10,
    heartbeat_interval_secs: 30,
}
```

### 1.3 uhorse-hub

云端 Hub 组件。

**核心功能**：
- 多节点管理
- 任务调度与路由
- 负载均衡
- 健康监控
- 统计信息收集

**配置**：
```rust
HubConfig {
    hub_id: "hub-001",
    bind_address: "0.0.0.0",
    port: 8765,
    max_nodes: 100,
    heartbeat_timeout_secs: 30,
    task_timeout_secs: 300,
    max_retries: 3,
}
```

## 2. 通信协议

### 2.1 连接生命周期

```
Node                              Hub
  │                                │
  │──── Register ─────────────────>│
  │<─── RegisterAck ───────────────│
  │                                │
  │──── Heartbeat ────────────────>│ (每 30 秒)
  │<─── HeartbeatAck ──────────────│
  │                                │
  │<─── TaskAssignment ────────────│
  │──── TaskAccepted ─────────────>│
  │                                │
  │──── TaskProgress ─────────────>│
  │                                │
  │──── TaskCompleted ────────────>│
  │                                │
  │<─── HeartbeatRequest ──────────│ (按需)
  │──── Heartbeat ────────────────>│
```

### 2.2 任务执行流程

```
用户                Hub                 Node
  │                  │                    │
  │── 提交任务 ─────>│                    │
  │                  │── TaskAssignment ─>│
  │                  │<── TaskAccepted ───│
  │                  │                    │
  │                  │<── TaskProgress ───│
  │                  │<── TaskProgress ───│
  │                  │<── TaskCompleted ──│
  │<── 任务结果 ─────│                    │
```

### 2.3 优先级系统

| 优先级 | 值 | 使用场景 |
|--------|---|----------|
| Background | 0 | 清理、维护任务 |
| Low | 1 | 批量处理 |
| Normal | 2 | 常规任务 |
| High | 3 | 重要任务 |
| Urgent | 4 | 紧急任务 |
| Critical | 5 | 系统关键任务 |

## 3. 安全机制

### 3.1 JWT 认证

```rust
// 创建认证器
let authenticator = NodeAuthenticator::with_secret("jwt-secret")?;

// 认证节点
let auth_info = authenticator.authenticate_node(&node_id, "credentials").await?;

// 验证令牌
let verified = authenticator.verify_token(&token).await?;

// 刷新令牌
let new_tokens = authenticator.refresh_token(&refresh_token).await?;
```

### 3.2 敏感操作审批

```rust
// 创建审批器
let approver = SensitiveOperationApprover::new(approval_manager);

// 检查是否需要审批
if approver.requires_approval("file_delete") {
    // 请求审批
    let request_id = approver.request_approval(
        &node_id,
        "file_delete",
        ApprovalLevel::Single,
        context,
    ).await?;

    // 等待审批
    let status = approver.get_approval_status(&request_id).await?;
}

// 批准/拒绝
approver.approve(&request_id, "admin", Some("已批准")).await?;
approver.reject(&request_id, "admin", Some("风险过高")).await?;
```

### 3.3 字段加密

```rust
// 创建加密器
let encryptor = HubFieldEncryptor::with_master_key(master_key);

// 加密数据
let encrypted = encryptor.encrypt(b"敏感数据")?;

// 解密数据
let decrypted = encryptor.decrypt(&encrypted)?;

// 加密 JSON
let encrypted_json = encryptor.encrypt_json(&json_data)?;
let decrypted_json: Value = encryptor.decrypt_json(&encrypted_json)?;
```

### 3.4 TLS 配置

```rust
// 创建 TLS 配置
let tls_config = HubTlsConfig::new("/path/to/cert.pem", "/path/to/key.pem");

// 在 Hub 中使用
let manager = SecurityManager::new("jwt-secret", approval_manager)?
    .with_tls("/etc/ssl/cert.pem", "/etc/ssl/key.pem");
```

## 4. 配置管理

### 4.1 Hub 配置文件

`hub.toml`:

```toml
[hub]
id = "hub-001"
bind_address = "0.0.0.0"
port = 8765
max_nodes = 100
heartbeat_timeout_secs = 30
task_timeout_secs = 300
max_retries = 3

[security]
jwt_secret = "your-secret-key"
tls_cert = "/etc/ssl/cert.pem"
tls_key = "/etc/ssl/key.pem"

[storage]
type = "sqlite"
path = "/var/lib/uhorse/hub.db"

[logging]
level = "info"
format = "json"
output = "/var/log/uhorse/hub.log"
```

### 4.2 Node 配置文件

`node.toml`:

```toml
[node]
id = "node-001"
name = "开发节点"
hub_url = "ws://localhost:8765"
max_concurrent_tasks = 10
heartbeat_interval_secs = 30

[workspace]
name = "main"
path = "/home/user/workspace"
read_only = false
allowed_patterns = ["*"]
denied_patterns = []

[logging]
level = "info"
output = "/var/log/uhorse/node.log"
```

### 4.3 环境变量

```bash
# Hub 配置
UHORSE_HUB_ID=hub-001
UHORSE_HUB_BIND=0.0.0.0
UHORSE_HUB_PORT=8765

# Node 配置

UHORSE_NODE_NAME="开发节点"
```

## 5. API 参考

### 5.1 Hub HTTP API

#### 节点管理

```
GET    /api/v1/nodes              # 获取所有节点
GET    /api/v1/nodes/:id          # 获取节点详情
POST   /api/v1/nodes              # 注册节点
DELETE /api/v1/nodes/:id          # 注销节点
```

#### 任务管理

```
GET    /api/v1/tasks              # 获取所有任务
GET    /api/v1/tasks/:id          # 获取任务详情
POST   /api/v1/tasks              # 提交任务
DELETE /api/v1/tasks/:id          # 取消任务
```

#### 统计信息

```
GET    /api/v1/stats              # 获取系统统计
GET    /api/v1/health             # 健康检查
```

### 5.2 WebSocket 协议

WebSocket 端点：`ws://hub-host:8765/ws`

#### 节点 → Hub 消息

| 消息类型 | 说明 |
|---------|------|
| `Register` | 节点注册 |
| `Heartbeat` | 心跳 |
| `TaskAccepted` | 任务已接受 |
| `TaskProgress` | 任务进度 |
| `TaskCompleted` | 任务完成 |
| `TaskFailed` | 任务失败 |

#### Hub → 节点消息

| 消息类型 | 说明 |
|---------|------|
| `RegisterAck` | 注册确认 |
| `HeartbeatAck` | 心跳确认 |
| `TaskAssignment` | 任务分配 |
| `TaskCancel` | 取消任务 |

### 5.3 消息格式

所有消息使用二进制格式，包含：
- 消息头 (4 字节长度 + 1 字节类型)
- 消息体 (JSON 或二进制)

提供预置仪表板：

- 节点状态
- 任务统计
- 性能分析

### 6.3 告警规则

示例告警规则：

```yaml
groups:
  - name: uhorse
    rules:
      - alert: NodeOffline
        expr: uhorse_nodes_total{status="online"} < 1
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "没有在线节点"

      - alert: HighTaskFailureRate
        expr: rate(uhorse_tasks_total{status="failed"}[5m]) > 0.1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "任务失败率过高"
```

```bash
# 查看任务详情

tail -f /var/log/uhorse/node.log
```

## 8. 最佳实践

### 8.1 部署建议

- Hub 部署在稳定的服务器环境
- Node 部署在用户本地机器
- 使用 TLS 加密通信
- 配置心跳间隔为 30 秒
- 设置合理的任务超时

### 8.2 性能优化

- 根据机器性能调整 `max_concurrent_tasks`
- 使用 SSD 存储数据库
- 合理配置任务优先级
- 监控资源使用情况

### 8.3 安全建议

- 启用 JWT 认证
- 使用 TLS 加密
- 定期轮换密钥
- 审计敏感操作
- 限制网络访问
