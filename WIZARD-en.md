# uHorse Configuration Wizard Guide

## Overview

uHorse provides an interactive configuration wizard to help you quickly complete project configuration without manually editing configuration files.

## Launch Configuration Wizard

### Build Project

```bash
cargo build --release
```

### Run Configuration Wizard

```bash
# Run configuration wizard in current directory
./target/release/uhorse wizard

# Run configuration wizard in specified directory
./target/release/uhorse wizard -d /path/to/project

# Non-interactive mode (use defaults)
./target/release/uhorse wizard --non-interactive
```

---

## Configuration Wizard Flow

The wizard will guide you through the following steps:

### 1. Server Configuration

```
🌐 Server Configuration
─────────────────────────
? Server host (default: 127.0.0.1): 0.0.0.0
? Server port (default: 8080): 8080
? Enable TLS? (y/N): n
```

### 2. Database Configuration

```
💾 Database Configuration
─────────────────────────
? Database type:
  > SQLite (recommended for development)
    PostgreSQL (recommended for production)

? Database path (default: ./data/uhorse.db): ./data/uhorse.db
```

For PostgreSQL:
```
? PostgreSQL URL: postgresql://user:password@localhost/uhorse
? Connection pool size (default: 10): 10
```


## Command Line Options

```bash
uhorse wizard [OPTIONS]

Options:
  -d, --dir <PATH>           Target directory (default: current)
  -c, --config <FILENAME>    Config filename (default: config.toml)
  -n, --non-interactive      Use default values
  -f, --force                Overwrite existing config
  -h, --help                 Show help message
```

## Environment Variables

The wizard can read default values from environment variables:

| Variable | Description |
|----------|-------------|
| `UHORSE_HOST` | Server host |
| `UHORSE_PORT` | Server port |
| `DATABASE_URL` | Database URL |
| `OPENAI_API_KEY` | OpenAI API key |
| `ANTHROPIC_API_KEY` | Anthropic API key |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token |
| `SLACK_BOT_TOKEN` | Slack bot token |
| `JWT_SECRET` | JWT secret key |

### Example

```bash
export OPENAI_API_KEY="sk-..."
export TELEGRAM_BOT_TOKEN="123456789:ABC..."
export JWT_SECRET="my-secret-key"

./target/release/uhorse wizard
# Will use environment variables as defaults
```

## Generated Configuration

The wizard generates a `config.toml` file:

```toml
# Generated by uHorse Configuration Wizard
# Date: 2025-03-04

[server]
host = "0.0.0.0"
port = 8080

[database]
type = "sqlite"
path = "./data/uhorse.db"

[channels]
enabled = ["telegram"]

[channels.telegram]
bot_token = "123456789:ABC..."

[llm]
provider = "openai"

[llm.openai]
api_key = "sk-..."
model = "gpt-4"
temperature = 0.7

[security]
device_pairing = true
approval_workflow = true
audit_log = true

[logging]
level = "info"
format = "json"
```

## Next Steps

After configuration:

```bash
# Start server
uhorse run

# Or with specific config
uhorse run --config config.toml

# Verify health
curl http://localhost:8080/health/live
```

## Troubleshooting

### Permission Denied

```bash
# Make sure binary is executable
chmod +x ./target/release/uhorse
```

### Config Already Exists

```bash
# Force overwrite
uhorse wizard --force
```

### Missing Dependencies

```bash
# Check dependencies
uhorse doctor

# Fix issues
uhorse doctor --fix
```
